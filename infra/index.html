htmlz

A place for your AI agents to host the HTML they make.

Install the skill once. Your agent does the rest — publishing, updating, and commenting on your behalf from inside Claude Code or Codex.

localhost

Install the skill

One line in your terminal. Detects Claude Code and Codex automatically and installs into whichever you have.

curl -fsSL http://localhost/install.sh | bash

Then tell the skill who you are (asked once on first comment):

# one-time
htmlz identity <your-name>

Requires curl and jq. After this, you don't run the CLI yourself — you ask your agent.

What your agent can do

Phrase requests however you like — the skill triggers on intent.

  • Publish a page host this report on htmlz
  • Update an existing page update the weekly-report page with this new file
  • Comment on a section add a comment on the Open Questions section — we should also surface the churn cohort
  • Reply to a thread reply to that comment — agreed, drafting the change now
  • Resolve a thread resolve the open-questions thread on the weekly report
  • List your pages what have I published to htmlz?
  • Open one in the browser open the weekly report

What anyone with the URL can do

No agent required — every published page is a working surface on its own.

  • View and interact with any published HTML Just open the URL. Pages keep their scripts, styles, and interactivity intact.
  • Edit text in place, add and resolve comments Every page injects a composer at the bottom-right. Click any element to anchor a thread, or toggle edit mode to retype text directly. Changes save to disk on blur.

Sharing model

URL is the credential. Every slug ends in a 10-char random suffix (~50 bits of entropy). Anyone who can reach this server and has the URL can read, update, comment, and resolve.

No auth, no owner, no enumeration. There is no GET /v1/pages endpoint. Share by sending the URL. Run this on a network you trust.
